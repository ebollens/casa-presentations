Payload Representations

Two representations of an app

TransitPayload
Machine-readable UUID attribute names
Original and journalled attributes

LocalPayload
Local human-readable attribute names
Squashed attributes

Payload Attributes

Attribute specification by UUID

Defines payload operation mechanics:

Filter
Transform
Journal

Can be marked optional or required

Publisher Module

Respond to
GET /payloads

For each payload to be shared:

AdjOutTranslate
(MUST, if LocalPayload representation)

As simple as publishing a JSON file

Receiver Module

Data path for requests to
GET /payloads

For each payload received and validated:

AdjInTranslate
(MUST)
AdjInSquash
(MUST)
AdjInFilter
(MUST base, MAY extend)
AdjInStore
(MAY)

Relay Module

Data path for payloads
from receiver to publisher

For each payload through Receiver Module:

AdjOutTransform
(MUST base, MAY extend)
AdjOutStore
(MAY)
AdjOutFilter
(MUST base, MAY extend)

Deliver non-originating payloads to Publisher Module

Local Module

Respond to outlets with payloads

GET /local/payloads
(MUST)
GET /local/payloads/:org_id/:app_id
(MUST)
GET /local/payloads?search=TXT
(SHOULD)
GET /local/payloads/_:method?body=QRY
(SHOULD)

Also how to send originating payloads to Publisher Module

Manager Module

/admin/local/payloads/..
/admin/attributes/..

/admin/peers/out/..
/admin/settings

Reference Implementation

Configurability

Adapter-driven persistence & indexing

At launch, reference implementation supports:

Persistence - MySQL, MsSQL, Sqlite
Indexer - ElasticSearch

This is intra-AS knowledge

GET /local/payloads?search=TXT  (SHOULD)
GET /local/payloads/_:type?body=QRY  (SHOULD)

Scalability

ElasticSearch supported for  GET /local/payloads
and soon for  GET /payloads

Highly recommended for horizontal scaling
Caveat is eventual consistency

Trivial to add different index or persistence

Attribute Extensibility

Define UUID, section and behaviors

module CASA
  module Attribute
    class Title < Definition
      uuid '1f2625c2-615f-11e3-bf13-d231feb1dc81'
      section 'use'
      squash Strategy::Squash::String
      filter Strategy::Filter::String
      transform Strategy::Transform::String
    end
  end
end
